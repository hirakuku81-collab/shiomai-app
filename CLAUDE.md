# 塩梅アプリ プロジェクト

## アプリの概要
料理の塩分濃度を計算し、「よい塩梅（美味しい塩加減）」を管理するiPhone向けWebアプリ。
ファイル構成はシンプルに `index.html` 1ファイルで動作する。

## 開発者について
- プログラミング未経験。日本語で丁寧に説明すること
- 専門用語を使う場合は必ず補足を加える
- 手順は番号付きで、1ステップずつ説明する
- エラーが出た場合は原因と解決策をセットで伝える

## 回答スタイル
- 回答は必ず日本語
- コードの変更箇所は「どこを」「なぜ」変えるか説明してから実行する
- 大きな変更の前には必ず確認を取る

## 現在の機能（v3完成済み）
1. **塩分計算** — 食材重量・水分量・調味料から塩分濃度を計算し、一般的な適正値と比較・改善提案
2. **調味料ライブラリ** — 25種類のプリセット（醤油・味噌・魚醤・粉末だし・料理酒など）＋手動登録
3. **レシピ保存・呼び出し** — 計算結果をレシピとして保存、手順メモ・アレンジ編集、検索機能
4. **マイ塩梅学習** — 「薄い／ちょうどいい／濃い」の評価を蓄積し、料理タイプ別のマイ適正塩分%を自動算出

## データ保存
- レシピと評価履歴は `localStorage` に保存（ブラウザ内永続化）
- 将来的にはサーバーやFirebaseへの移行を検討

## 次に追加したい機能（優先順）
1. **カメラ撮影→OCR読み取り** — 調味料ボトルの栄養成分表示をカメラで撮影し、塩分濃度を自動入力
   - iOS Safari の `<input type="file" accept="image/*" capture="environment">` を起点に実装
   - OCRはGoogle Cloud Vision API または Tesseract.js を検討
   - 読み取った塩分量（per 100g）から濃度（%）を自動計算してライブラリに登録

2. **マイ塩梅からのレシピ提案** — 蓄積した好み（マイ適正値）を元に、次回同じ料理を作るときの調味料量を自動提案

3. **データエクスポート** — レシピ一覧をCSVまたはPDFで書き出す機能

## 安全ルール
- ファイル削除は必ず確認を取ること
- index.html を変更する前にバックアップを作成すること（index.html.bak）
- localStorage のデータを消す操作は絶対に確認を取ること

## 技術仕様
- 純粋なHTML/CSS/JavaScript（フレームワーク不使用）
- Tabler Icons を CDN から読み込み（アイコン用）
- iPhone Safari での動作を最優先（max-width: 390px）
- ダークモード対応（CSS変数で管理）
- 外部APIキーが必要な機能を追加する場合は .env ファイルで管理すること
